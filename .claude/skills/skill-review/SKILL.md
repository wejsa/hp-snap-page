# skill-review: 코드 리뷰

## 실행 조건
- 사용자가 `/skill-review {경로}` 또는 "{경로} 코드 리뷰해줘" 요청 시

## 실행 플로우

### 1. 리뷰 대상 파일 수집
단일 파일, 디렉토리, 패턴(`src/**/*Service.kt`) 지원

### 2. 코드 분석
파일 구조 파악, 의존성 확인, 테스트 파일 매칭

### 3. @agent-reviewer 5관점 검토

**도메인 체크리스트 동적 로딩**:
1. project.json에서 도메인 확인
2. `_base/checklists/*.md` (공통) + `{domain}/checklists/*.md` (도메인별) 로드
3. 중복 항목은 도메인 설정 우선

#### 5관점 체크 항목
| 관점 | 핵심 체크 |
|------|----------|
| 1. 컴플라이언스 | 규정 준수, 감사 로그, 민감정보 암호화, 데이터 보존 |
| 2. 도메인 | 비즈니스 로직, 상태 머신, 데이터 일관성, 도메인 모델 |
| 3. 아키텍처 | 설계 패턴, 장애 격리, 확장성, 의존성 |
| 4. 보안 | 인증/인가, 입력 검증, 민감정보 노출, Rate Limiting |
| 5. 테스트 | 커버리지 80%+, 실패 케이스, 동시성, 통합 테스트 |

### 4. 이슈 분류

| 심각도 | 설명 | 조치 |
|--------|------|------|
| 🔴 CRITICAL | 보안 취약점, 데이터 손실 | 즉시 수정 필수 |
| 🟠 MAJOR | 기능 오류, 성능 문제 | 수정 권장 |
| 🟡 MINOR | 코드 품질, 컨벤션 | 개선 권장 |
| 🔵 INFO | 제안, 참고사항 | 선택적 |

### 5. 리뷰 결과 출력

필수 포함: 리뷰 대상(파일 수/라인), 5관점 요약 테이블(관점/상태/이슈 수), 심각도별 이슈 목록([ID] 파일:라인-제목/관점/설명/수정방안), 개선 권장사항, 결론(승인 가능여부/필수·권장 수정 수)

## 자동 수정 제안
CRITICAL/MAJOR 이슈에 Before/After 코드 제안

## 에이전트 호출
복잡한 리뷰 시 `@agent-reviewer` 또는 프로젝트별 커스텀 리뷰어

## 주의사항
- 리뷰는 코드 수정 없이 분석만 수행
- CRITICAL 이슈 발견 시 즉시 알림
- 자동 수정은 사용자 승인 후에만 적용
